`@playbykey/theory` covers the computation layer for music theory - scales, modes, intervals, and key relationships. Zero dependencies, no external API calls, and no runtime overhead. Drop it into any JavaScript or TypeScript project and start computing scale notes, resolving intervals, and navigating key relationships immediately.

## Installation

```bash
npm install @playbykey/theory
# or
pnpm add @playbykey/theory
# or
yarn add @playbykey/theory
```

## Quickstart

```typescript
import { getScaleNotes, getParentScaleModes } from '@playbykey/theory';

// Get all notes in D Dorian
const notes = getScaleNotes('D', 'dorian');
console.log(notes); // ['D', 'E', 'F', 'G', 'A', 'B', 'C']

// Find the parent major key
const parentModes = getParentScaleModes('D', 'dorian');
const parent = parentModes.find(m => m.mode === 'ionian');
console.log(parent?.root); // 'C'
```

## TypeScript

The package is written in TypeScript with strict mode. All exported functions are fully typed. No `any`. Import types directly:

```typescript
import type { Note, ModeName } from '@playbykey/theory';
```

## Note notation

The package uses sharps-only notation (`C#`, not `Db`). Reference `ENHARMONIC_LABELS` for display-friendly flat/sharp labels.
