Every constant below is exported from `@playbykey/theory`. The catalog covers the 12 chromatic `NOTES`, the 7 diatonic `MODES`, the full `INTERVAL_DEFINITIONS` map, `SCALE_DEFINITIONS` for derived scale types, and `NOTATION_IDS` for display mapping. Select inputs to see live values and the exact function call.

## Notes

`NOTES` is the ordered list of all 12 chromatic notes. `ENHARMONIC_LABELS` maps each accidental note to its flat/sharp display string.

<NotesExplorer client:load />

## Modes

`MODES` describes each of the 7 diatonic modes — name, scale degree, and character. `MODE_INTERVALS` and `MODE_SEMITONE_OFFSETS` give the step sizes and absolute offsets from the root.

<ModeExplorer client:load />

## Intervals

`INTERVAL_DEFINITIONS` maps each interval ID to a label and semitone count. `resolveIntervalEndpoints` uses this data to find the actual notes that bound an interval in a given key and mode. See the [Intervals](/theory/intervals/) page for live resolution and the full type reference.

<IntervalExplorer client:load />

## Scales

`SCALE_DEFINITIONS` maps each scale kind to a label and how its notes are computed. `getScaleNotes` uses this to return the correct note set for any root and scale type.

<ScalesExplorer client:load />

## Notation and display

`buildNoteMap(root, scaleType)` returns one entry per in-scale note with `note`, `scaleDegree` (always a number), and `isRoot`. Notation is left to the consumer — use `entry.note` for letter labels or `String(entry.scaleDegree)` for numeric labels.
